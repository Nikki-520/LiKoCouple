# LiKo 设计文档

> 服务于情侣两人的私密静态网站
> 纯前端 SPA | Vue 3 + Vite | 本地存储 | 治愈手绘风

---

## 一、项目概述

**LiKo** 是一个为情侣二人打造的私密在线空间，承载日常记录、回忆整理、互动娱乐等功能。网站定位为"两个人的秘密花园"，以治愈手绘风格为主调，强调仪式感、温暖感和丰富细腻的交互体验。

### 核心定位

- **私密**：仅服务两人，浏览内容无需登录，发表内容时可选身份登录
- **治愈**：视觉风格温暖柔和，手绘质感，低攻击性
- **有生命力**：处处可交互、处处有反馈，页面像活的一样
- **轻量**：纯前端静态网站，零服务器依赖

## 二、技术选型

| 维度 | 选择 | 说明 |
|------|------|------|
| 框架 | Vue 3 (Composition API) | 响应式数据 + 组件化 |
| 构建工具 | Vite | 快速开发与打包 |
| 路由 | Vue Router 4 | SPA 页面路由 |
| 动画 | CSS Animations/Transitions + GSAP | 翻书 3D 动画使用 GSAP，其余手写 CSS |
| 数据存储 | localStorage | 纯前端存储，后期可迁移至云数据库 |
| UI 框架 | 无 | 纯手工 CSS，保证风格独特性 |
| 图标 | 自制 SVG / 手绘风格图标 | 配合治愈手绘风 |
| 字体 | 优先选用圆体/手写体 | 后续设计阶段确定具体字体 |

## 三、视觉设计

### 3.1 整体风格

**治愈手绘风** — 仿水彩/蜡笔质感，温暖色调，圆润 UI 组件，手工感插画元素，像一本精致的纸质手账。

### 3.2 配色方案：蜜桃奶霜

| 角色 | 色值 | 用途 |
|------|------|------|
| 主色 | `#FFE4D6`（奶油粉） | 页面背景、大区块底色 |
| 点缀色 A | `#FFAB76`（蜜桃橘） | 按钮、强调元素、标题装饰 |
| 点缀色 B | `#B5EAEA`（薄荷绿） | 辅助色、完成状态、清新点缀 |
| 中性色 | `#FFF5F0`（暖白） | 卡片背景、内容区 |
| 文字色 | `#5C4A3D`（暖棕） | 正文文字，避免纯黑过于生硬 |
| 辅助文字 | `#A8988A`（浅棕灰） | 次要文字、日期标注 |

### 3.3 视觉元素

- **圆角**：所有卡片、按钮使用大圆角（12px-20px），营造柔和感
- **阴影**：使用柔和的彩色投影（带主色调的半透明阴影），非纯黑阴影
- **纹理**：背景可有轻微纸张纹理/噪点，增强手工感
- **插画**：手绘风格的小元素——云朵、花朵、纸飞机、小星星、信封、相机、音符等
- **字体方向**：标题用圆体/艺术字，正文用清爽无衬线，强调温暖易读

## 五、首页 Hub 设计

### 5.1 翻书动画（进入体验）

**流程：**
1. 进入网站即展示手绘风日记本封面，中央写有 "LiKo"，周围点缀小插画（云、花、纸飞机）
2. 用户点击封面或等待 2 秒后自动触发翻书动画
3. 封面从右向左 3D 翻转（GSAP + CSS 3D transforms），露出扉页
4. 扉页短暂停留 2-3 秒后自动轻柔翻到 Hub 首页
5. 配柔和纸张翻页音效（可选，CSS `@media (prefers-reduced-motion)` 优雅降级）
6. 非首次访问时（sessionStorage 有标记），跳过翻书动画直接进入 Hub

**技术要求：**
- CSS `perspective` + `rotateY` 实现 3D 翻书
- GSAP Timeline 编排动画序列
- 翻页时封面背面有纸张纹理阴影

### 5.2 Hub 首页布局

**背景：** 奶油粉渐变色底，漂浮手绘小元素（CSS 动画驱动花瓣/光点/小星星缓缓游动）

**顶部区域：**
- 右上角：登录/身份按钮（未登录显示「💕 登录」，已登录显示当前角色昵称，点击可切换身份或退出）
- 「我们在一起」天数大数字（基于纪念日自动计算）
- 下一纪念日倒计时（小字提示）

**卡片区域（8 张入口卡片）：**
- 2 列 × 4 行网格布局（移动端 1 列）
- 每张卡片包含：手绘小图标 + 模块名称 + 一行小描述
- 卡片有轻微纸质纹理底色、圆角、柔和彩色阴影

**底部：** 微小版权文字或暖心小语

### 5.3 卡片入场动画

翻书完成后，8 张卡片依次从画面外飞入：
- 使用 `stagger` 延迟（每张间隔 100-150ms）
- 每张卡片带有弹性缓出（`cubic-bezier` 或 spring 曲线）
- 方向混合——左上两张从左飞入，右上两张从右飞入，下方依次上浮
- 像被一只看不见的手把卡片一张张摆到桌面上

### 5.4 卡片 hover 交互

- 微微放大（scale: 1.03 → 1.05）
- 阴影扩大变柔和
- 图标小幅弹跳/摇摆（独立动画）
- 鼠标移入有圆形光晕跟随

## 六、自定义鼠标 & 拖尾特效

### 6.1 自定义鼠标样式

- 隐藏系统默认光标（`cursor: none` 在自定义区域）
- 用跟随鼠标的 div 替代，样式为柔和的小圆形/花瓣形
- 在不同情境下变换样式：
  - 默认：柔和小圆点（带淡粉色发光）
  - 可点击元素上：变为心形或小手图形
  - 文本编辑区：变为羽毛笔/铅笔图形

### 6.2 鼠标拖尾

- 鼠标移动时后方留下 5-10 个拖尾粒子
- 粒子为小花瓣/光点形状
- 粒子逐渐缩小、降低透明度、向外微微扩散
- 使用 `requestAnimationFrame` 驱动位置更新
- 粒子通过 CSS transition 或 JS 手动更新 opacity/scale 实现消散

### 6.3 实现方案

```text
核心思路：在全局挂载一个 canvas 或一组绝对定位的 div 元素
- 鼠标位置记录在 reactive 变量中
- 每帧将新粒子推入数组，过期粒子自动移除
- 所有粒子用 v-for 渲染，CSS transition 处理消散
- 性能优化：粒子数量限制在 20 个以内
```

## 七、可选身份登录系统

### 7.1 核心理念

登录是**可选的**——任何人都可以直接浏览 LiKo 的所有内容（日记、相册、留言等），不需要登录。但当你想要**发表内容**（写日记、留言、添加照片等）时，需要先选择一个身份登录，让系统知道这条内容是谁留下的。

### 7.2 预设角色

系统中预设两个默认角色：

- **角色 A（默认男方视角）**：昵称可自定义（默认"他"），专属密码
- **角色 B（默认女方视角）**：昵称可自定义（默认"她"），专属密码

角色信息存储在 localStorage 中，可随时修改昵称和密码。

### 7.3 登录入口

**三个触发入口：**

1. **右上角按钮**：所有页面右上角始终显示登录按钮
   - 未登录：显示「💕 登录」
   - 已登录：显示当前角色昵称（如「小明」），点击可切换身份或退出

2. **操作拦截弹窗**：当用户尝试以下操作且未登录时，自动弹出登录框：
   - 写日记 / 编辑日记
   - 添加照片
   - 新增愿望
   - 写留言
   - 答题
   - 添加音乐
   - 添加回忆碎片

3. **手动触发**：用户主动点击右上角按钮

### 7.4 登录弹窗设计

- 温馨小弹窗（Modal），背景玻璃拟态，不打断当前页面感受
- 标题：「选择你的身份」
- 两个角色卡片并排：
  - 左侧卡片：男方头像/图标 + 昵称
  - 右侧卡片：女方头像/图标 + 昵称
- 点击角色 → 卡片翻转/展开，露出密码输入框
- 输入密码 → 登录成功 → 弹窗关闭
- 首次访问 → 引导设置昵称和密码

### 7.5 交互流程

```
用户触发发表操作（或点击右上角登录）
       ↓
  已登录？
   ├── 是 → 直接操作，自动标记作者
   └── 否 → 弹出登录弹窗
              ↓
         首次使用？
          ├── 是 → 引导设置双方昵称和密码
          └── 否 → 展示两个角色卡片
                     ↓
                点击自己的角色
                     ↓
                输入密码
                     ↓
                验证通过？
                 ├── 否 → 输入框微震 + 提示「再想想哦～」
                 └── 是 → 角色头像闪现爱心
                          → 登录状态存入 sessionStorage
                          → 弹窗关闭，继续执行之前的操作
```

### 7.6 当前身份追踪

- 登录后，`sessionStorage` 记录当前角色 ID（`A` 或 `B`）和昵称
- 右上角显示当前昵称，点开可看到「切换身份」和「退出登录」
- 发表的所有内容自动标记作者角色 + 昵称
- 切换身份 = 退出当前 + 弹出登录弹窗选另一角色
- 关闭浏览器后 sessionStorage 清空，再次访问又回到未登录状态（仍可浏览）

### 7.7 存储

- `liko_auth` — `{ personA: {name, pass}, personB: {name, pass} }` 存入 localStorage
- `liko_session` — `{ role: 'A'|'B', name }` 存入 sessionStorage（当前会话身份）
- 首次使用引导设置时，两次设置互不干扰，一个人设置完另一个人可随时设置

## 八、功能模块详细设计

### 8.1 时光日记 (`/diary`)

**功能：**
- 写日记：日期选择器 + 心情选择（😊😢😍😴 表情图标可点击切换）+ 正文编辑 + 图片上传（Base64 存 localStorage）
- 日记列表：时间线卡片展示，点击展开全文
- 编辑/删除已有日记

**特效：**
- 写日记时自定义光标变为羽毛笔样式
- 保存时纸团揉捏 → 展开变平整动画（隐喻"把心事写下来"）
- 心情图标切换带弹性动画
- 新日记卡片以飞入/绽放效果出现
- 删除时纸片粉碎/飘散动画

### 8.2 回忆相册 (`/album`)

**功能：**
- 照片宫格/瀑布流排列
- 点击单张照片进入全屏浏览模式
- 可按时间或自定义标签筛选
- 支持新增照片（Base64）+ 添加描述

**特效：**
- 照片 hover 像拍立得微微摇晃（rotate ±2deg + translateY）
- 点击照片时带有「从相册中抽出」的过渡动画
- 全屏浏览模式下左右滑动像翻相册页（swipe 或按钮驱动）
- 照片加载时从模糊到清晰的渐显效果

### 8.3 纪念日 (`/memorial`)

**功能：**
- 在一起天数大数字（醒目展示、实时更新）
- 重要日期时间线（在一起那天、第一次旅行、生日……可自行添加）
- 下一纪念日倒计时环形进度条

**特效：**
- 天数数字像翻牌钟一样逐个翻动（数字翻转动画）
- 倒计时环带心跳脉动效果（scale 周期性变化）
- 纪念日当天触发庆祝特效：彩色粒子从屏幕底部升腾
- 时间线节点带有柔光扩散动画

### 8.4 愿望清单 (`/wishlist`)

**功能：**
- 新增愿望（标题 + 描述）
- 完成勾选：文字加删除线 + 淡色
- 未完成愿望展示为挂在绳子上的卡片

**特效：**
- 未完成卡片微微摇摆（像挂在绳子上被风吹动）
- 勾选完成时出现「盖章」动画 + 星星粒子四散飞出
- 已完成项收到底部「已实现」区域，有归档的满足感动画
- 拖拽可调整顺序（轻度实现）

### 8.5 留言板 (`/messages`)

**功能：**
- 展示两人互相留下的留言/小纸条
- 新增留言：选择收件人 + 内容
- 留言以信封/便签样式呈现
- 标记已读/未读

**特效：**
- 新留言就像小信封飞入信箱
- 未读信封微微发光提示
- 点击信封 → 拆封动画（封口撕开 → 信纸抽出）
- 便签贴在软木板上的 3D 斜贴效果
- hover 时便签一角微微掀起（rotate + shadow）

### 8.6 默契问答 (`/quiz`)

**功能：**
- 预设题库（或自定义出题）
- 两个人都答完后揭晓答案，显示默契度百分比 + 星级

**特效：**
- 题目以卡片翻转方式出现
- 选择答案时按钮有按压感
- 结果页爱心粒子从中央爆炸/填满
- 默契度以进度条动画 + 星星逐个点亮展示
- 高分时有撒花庆祝动画

### 8.7 专属音乐盒 (`/music`)

**功能：**
- 预设两人的专属歌单，可添加歌曲链接/名称
- 播放/暂停控制
- 音量调节

**特效：**
- 播放时黑胶唱片旋转（CSS animation）
- 音符粒子从唱片中心飘出，向上升腾消散
- 唱片周围有随音乐脉动的柔和光晕/氛围灯
- 暂停时唱片缓慢停止（带减速缓出）

### 8.8 回忆碎片 (`/fragments`)

**功能：**
- 随机展示一条过去的记录/情话/照片片段
- 用户自己添加碎片内容
- 翻牌式交互

**特效：**
- 碎片以卡牌形式展示（正面朝下）
- 点击翻牌：3D 旋转 180° 露出内容（正反面设计）
- 翻牌后金色/粉色粒子从卡面洒落
- 可「再翻一张」刷新随机内容
- 刷新动画：当前卡牌飞出 → 新卡牌飞入

## 九、动画系统设计

### 9.1 全局动画规范

| 动画类型 | 时长 | 缓动函数 | 用途 |
|----------|------|----------|------|
| 微交互 | 200-300ms | `cubic-bezier(0.4, 0, 0.2, 1)` | hover、点击反馈 |
| 入场动画 | 400-600ms | spring-like ease-out | 元素出现、页面切换 |
| 退出动画 | 250-400ms | ease-in | 元素消失、关闭 |
| 氛围动画 | 2-6s | ease-in-out / linear | 背景漂浮、粒子 |

### 9.2 页面切换过渡

- 首页 → 子页面：卡片放大飞入过渡
- 子页面 → 首页：缩小飞回卡片位置
- 子页面之间：淡入淡出 + 轻微位移动画

### 9.3 音效（可选）

- 所有音效默认关闭，可在设置中开启
- 翻书纸张音效
- 按钮轻点音效
- 完成/庆祝音效
- 使用 Web Audio API 或 `HTMLAudioElement`

## 十、数据存储方案

### 10.1 当前方案：localStorage

```text
localStorage 数据结构设计：

liko_auth          — 双人凭据 { personA: {name, pass}, personB: {name, pass} }
liko_diary         — 日记数组 [{id, date, mood, content, images[]}]
liko_album         — 照片数组 [{id, date, tags[], imageBase64, desc}]
liko_memorials     — 纪念日数组 [{id, date, title, type}]
liko_wishlist      — 愿望数组 [{id, title, desc, done, date}]

liko_quiz          — 问答记录 [{id, question, answerA, answerB, date}]
liko_playlist      — 歌单数组 [{id, title, url, artist}]
liko_fragments     — 碎片数组 [{id, content, type, date}]
liko_settings      — 设置 { soundEnabled: bool }
```

### 10.2 后期上云方案

- 保留 localStorage 作为离线缓存
- 引入简单的后端 API（如 Cloudflare Workers + D1 / Supabase）
- 数据同步策略：乐观更新 + 定期同步
- 数据模型不变，仅更换持久层

## 十一、响应式设计

- **Desktop（≥1024px）**：2 列卡片网格，全功能体验
- **Tablet（768-1023px）**：适当缩小卡片和间距
- **Mobile（<768px）**：单列布局，卡片堆叠，触摸手势优化
  - 翻书动画改为轻量的 fade + scale 过渡
  - 双人解锁并排改为上下排列
  - 自定义鼠标改为触摸涟漪效果

## 十三、设计参考/灵感方向（供 AI 出图）

### 首页封面插画描述

> 一本翻开的手绘风格日记本封面，温暖的奶油粉色背景上有轻柔的水彩质感。封面中央用手写体写着"LiKo"，周围点缀着可爱的简笔画小元素：蓬松的云朵、几朵小花、一架纸飞机从角落飞过。整体色调温暖柔和，像在午后阳光下的一本手工手账，纸张边缘有轻微的做旧纹理。背景有极淡的格子线，像真正的笔记本纸张。

### 入口卡片插画风格

> 8 张圆角卡片，每张有独特的简笔画图标：
> - 日记：一本翻开的小本子 + 羽毛笔
> - 相册：一台拍立得相机
> - 纪念日：两颗靠在一起的心 + 日历
> - 愿望清单：一个许愿瓶或星星
> - 留言板：一枚封信 + 小爱心
> - 问答：两个对话气泡重叠
> - 音乐：一个黑胶唱片 + 音符
> - 回忆碎片：一个打开的宝箱/小盒子
> 所有图标都是简洁的线条画风，带柔和填色，像用手绘板画出来的。

### UI 元素风格统一

> 按钮圆角充足，带微妙的渐变色，hover 时有毛茸茸的发光边缘。输入框像手工纸的便签条。所有元素在细微处有手工感的不完美——像真的画出来的，而非精确的像素对齐。
